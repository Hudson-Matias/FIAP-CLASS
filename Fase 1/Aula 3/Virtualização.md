A virtualização é uma tecnologia que permite criar uma camada de abstração sobre o hardware do computador, permitindo que os elementos de hardware de um único computador, como processadores, memória, armazenamento e muito mais, sejam divididos em vários computadores virtuais, comumente chamados de máquinas virtuais. Cada máquina virtual executa seu próprio sistema operacional e se comporta como um computador independente, embora esteja sendo executada em apenas uma parte do hardware do computador subjacente real.
## Componentes de virtualização: hypervisor
O hypervisor é um software que fornece uma camada de virtualização e abstrai o processador, a memória e a rede e o armazenamento do sistema de computação, permitindo a criação de várias máquinas virtuais.
O hypervisor é uma camada de software que reside abaixo das máquinas virtuais e acima do hardware.
O hypervisor gerencia as interações entre cada máquina virtual e o hardware compartilhado. Cada máquina virtual executa seu próprio sistema operacional, o que permite que vários sistemas operacionais sejam executados, simultaneamente, no mesmo sistema de computação físico. O hypervisor fornece recursos de hardware padronizados para todas as máquinas virtuais. Esse software possui dois componentes principais: o kernel e o monitor de máquina virtual (Virtual Machine Manager - VMM). Um kernel de hypervisor fornece a mesma funcionalidade que o kernel de qualquer sistema operacional, incluindo gerenciamento de processos, gerenciamento de sistema de arquivos e gerenciamento de memória. Esse hypervisor foi projetado e otimizado para executar várias máquinas virtuais ao mesmo tempo, para receber solicitações de recursos por meio do VMM e apresentar as solicitações ao hardware físico.
Cada máquina virtual recebe um VMM que obtém um compartilhamento do processador, da memória, dos dispositivos de I/O e do armazenamento do sistema de computação físico, para executar a máquina virtual com êxito. O VMM abstrai o hardware físico e aparece como um sistema de computação físico com: processador, memória, dispositivos de I/O e outros componentes essenciais para a execução de um SO e aplicativos. O VMM recebe solicitações de recursos da máquina virtual as quais são transmitidas ao kernel, apresentando o hardware virtual da máquina virtual.

- Bare Metal Hypervisor-Type 1: também conhecido como hypervisor tipo 1. Esse hypervisor é instalado diretamente, na parte superior do hardware do sistema, sem nenhum sistema operacional subjacente ou qualquer outro software. Esse tipo de virtualização foi projetado principalmente para data centers corporativos e para servidores. Exemplos de hypervisor desse tipo são: Oracle VM server for SPARC, VMWare ESXi e Hyper-V.
- Hosted Hypervisor-Type 2: também é chamado de hypervisor tipo 2- Esse hypervisor é instalado como um aplicativo ou software em um sistema operacional. Nessa abordagem, o hypervisor não tem acesso direto ao hardware e todas as solicitações devem passar pelo sistema operacional em execução no sistema de computação físico- Esse tipo de virtualização de computação é projetado principalmente para usuários de desktop- Dessa maneira, temos como exemplo de hypervisor desse tipo: VMware Fusion, VMware Workstation, Oracle VM VirtualBox.
Um dos benefícios apresentados pelos hypervisors tipo 2, é o de suportar uma grande variedade de hardware, justamente porque é executado em cima de um sistema operacional. Frequentemente os hypervisors tipo 2 são fáceis de instalar e implantar, pois grande parte do trabalho de configuração de hardware, como rede e armazenamento, já foi definida pelo sistema operacional.
Em contrapartida, os hypervisors tipo 2 não são tão eficientes quanto os hypervisors tipo 1, devido a essa camada extra entre o próprio hypervisor e o hardware. Além disso, são menos confiáveis, porque existem mais pontos de falha que podem afetar a disponibilidade do sistema operacional subjacente, e impactar o hypervisor e os sistemas operacionais convidados existentes. Por exemplo, os patches padrão do sistema operacional, que exigem a reinicialização do sistema, forçariam a reinicialização de todas as máquinas virtuais naquele host.
Uma implantação de hypervisor tipo 2 usa mais recursos físicos do que um hypervisor tipo 1, isso seguindo a perspectiva de que o sistema operacional subjacente também consome recursos do sistema além daqueles consumidos pelas atividades do hypervisor.
Os hypervisors tipo 2 são normalmente usados em ambientes de desenvolvimento de desktop, quando um desenvolvedor está trabalhando em uma ou mais máquinas virtuais, na construção de um aplicativo, ou até mesmo para quem deseja ter um ambiente de teste para realizar os procedimentos relacionados aos seus estudos, sem afetar diretamente o sistema operacional do host físico. Essa é uma observação muito importante, visto fizemos uso de máquinas virtuais para o desenvolvimento desta aula.
## Função de um hypervisor
![[Pasted image 20260409115612.png]]O  conceito é bem simples, mas de acordo com o PORTNOY (2016), deve atender a três características bem definidas frente a suas tarefas:
- Fornecer um ambiente idêntico ao ambiente físico.
- Fornecer esse ambiente com custo mínimo de desempenho.
- Reter o controle total dos recursos do sistema.
Para que muitos convidados (máquinas virtuais) compartilhem os recursos físicos (hardware) de um host, dois princípios são esperados. O primeiro é que o convidado precisa ver e ter acesso aos vários recursos de hardware para funcionar de maneira eficaz. O sistema operacional do convidado deve ser capaz de usar unidades de disco, acessar memoria e fazer uso dos recursos de rede.
Da perspectiva de um sistema operacional convidado, o que um hypervisor faz é "enganar" esse convidado e fazer com que ele acredite que pode realmente ver e interagir diretamente com os dispositivos físicos do host. Essa abstração de hardware é apresentada na figura "Hypervisor: Abstração de hardware". Na verdade, cada convidado utiliza apenas uma fração dos recursos do host físico. Vamos a um exemplo no qual um host tem 16 GB de memoria RAM física disponível; um convidado pode acreditar que tem 2 GB para seu uso e gravar arquivos em uma unidade de disco com 1 00 GB. Mas, na verdade, ele estará trabalhando apenas com uma parte de um sistema de arquivos também existente no host físico. Os recursos de processamento e de rede funcionam de maneira semelhante, em que um convidado pode ter duas CPUs virtuais e acesso a uma única placa de interface de rede (NIC). Lembre-se de que o host físico deverá ter muito mais recursos
que o convidado.
O segundo principio é que o hypervisor não só precisa abstrair o hardware de cada um dos convidados virtuais, mas também equilibrar essa carga de trabalho. Cada convidado faz demandas constantes nos vários subsistemas de recursos. O hypervisor deve atender a todas essas demandas, agindo como um intermediário entre cada convidado e os dispositivos físicos. Além disso, ele deve fornecer recursos oportunos e adequados a todos.
A virtualização é uma tecnologia disruptiva, que muda a ideia de como os computadores físicos são tratados e como os serviços são entregues. Para entender por que a virtualização teve um efeito tão profundo no ambiente de computação atual, precisamos compreender o que aconteceu no passado. A tecnologia de virtualização surge na década de 1960, porém passou
a ser adotada amplamente somente no inicio dos anos 2000. As tecnologias que tornaram a virtualização uma realidade, como os hypervisors, foram desenvolvidas há décadas para dar a vários usuários acesso simultâneo a computadores que realizavam processamento em lote. O processamento em lote era uma prática computacional popular no setor de negócios para a execução rápida de tarefas rotineiras que repetiam milhares de vezes (RED HAT, 2023).
A virtualização na geralmente refere-se à de algum componente físico em um objeto logico. Ao virtualizar um objeto, podemos obter alguma medida maior de utilidade do recurso que o objeto fornece, como nas LANs virtuais. Essas redes locais proporcionam um bom desempenho de rede e gerenciamento aprimorado por serem separadas do hardware físico.
Se você ainda não está familiarizado com a ideia de virtualização de computadores, inicialmente imagine algo relacionado à realidade virtual. Essa tecnologia pode oferecer uma experiência de, realmente, estar naquele ambiente virtual que foi criado. Em um nível fundamental, é disso do que se trata a virtualização de computadores: como um aplicativo de computador experimenta seu ambiente criado (PORTNOY, 2016).
Por definição, a virtualização de computador é uma máquina virtual (Virtual Machine, VM), que pode virtualizar todos os recursos de hardware, incluindo processadores, memória, armazenamento e conectividade de rede. Outra definição atrelada, é uma tecnologia que permite criar vários ambientes simulados ou recursos dedicados, a partir de um único sistema de hardware
físico.
## Benefícios Relacionados a Virtualização
Um dos propósitos das organizações de TI recorrerem à virtualização é a redução de custos. A virtualização proporciona um aumento na utilização dos recursos computacionais, uma redução do número de servidores físicos e dos requisitos de energia e refrigeração associados, e um uso mais eficiente do espaço físico do data center. Essa tecnologia também gera uma ampla gama de benefícios de despesas operacionais, como aumento da produtividade de TI e simplificação de tarefas de gerenciamento.
A virtualização permite níveis mais altos de disponibilidade, tornando mais fácil manter as cargas de trabalho em execução, em casos de paralisações dos serviços existentes de uma forma planejada ou não. Além disso, a virtualização oferece um nível de controle e flexibilidade que não seria possível com a utilização de recursos físicos dedicados.
A virtualização também oferece soluções para disponibilidade contínua em caso de falhas de hardware e outras causas de tempo de inatividade não planejada. Por exemplo, com um data center virtualizado, podemos implementar soluções de tolerância a falhas para fornecer disponibilidade contínua aos seus aplicativos de missão crítica, executando duas instâncias de
um aplicativo em lockstep. Os sistemas Lockstep são sistemas de computador tolerantes a falhas que executam o mesmo conjunto de operações ao mesmo tempo. Dessa forma, se houver um problema com a instância primária, a instância secundária continuará de onde a primária parou, fornecendo disponibilidade contínua para suas cargas de trabalho mais importantes.
Existem empresas especializadas no assunto de Fault Tolerance, como a Vmware. Essa empresa apresenta soluções, como o Symmetric Multiprocessing Fault Tolerance (SMP-FT), que fornece zero perda de dados e tempo de inatividade, tendo capacidade de recuperar uma máquina virtual
instantaneamente, continuando a trabalhar no caso de uma falha de hardware. Observe a figura "Symmetric Multiprocessing Fault Tolerance", e note que, se a primeira máquina virtual falhar, instantaneamente, a segunda máquina virtual entra em cena.
## Soluções de Virtualização
Várias empresas oferecem soluções de virtualização que abrangem tarefas específicas de Data Center ou cenários de virtualização de desktop focados no usuário final. Exemplos mais conhecidos incluem a VMware, ela é especializada em virtualização de servidor, desktop, rede e armazenamento. Já a Citrix tem um nicho em virtualização de aplicativos, mas também oferece
virtualização de servidor e soluções de desktop virtual. A Microsoft, por sua vez, apresenta solução de virtualização Hyper-V, com o Windows, e se concentra em versões virtuais de servidores e desktops. E, por fim, o famoso VirtualBox, que é um virtualizador completo de uso geral para hardware x86, direcionado para servidor, desktop e uso integrado. As ferramentas do ambiente de trabalho de virtualização que serão escolhidas dependerão de vários fatores, incluindo o que está sendo executado como um sistema operacional de desktop e suas aplicações. Os exemplos apresentados nesta aula usam VirtualBox, mas você pode ficar à vontade para escolher outras soluções.
Podemos instalar e executar quantas máquinas virtuais desejarmos, mas devemos lembrar que os limites práticos são os recursos de hardware existentes, como espaço em disco, memória etc.
