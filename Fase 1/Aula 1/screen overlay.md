Dentro de um RAT ou trojan bancário, **screen overlay** é uma **camada falsa colocada por cima da tela de um app legítimo** para enganar a vítima. Em vez de ver a interface real do banco, login ou carteira digital, a pessoa vê uma tela parecida criada pelo malware. A Microsoft descreve esse tipo de ataque como uma sobreposição invisível ou falsa sobre apps legítimos para interceptar o que o usuário digita.

Na prática, ele serve para coisas como:

- **roubar usuário, senha, PIN ou código**
    
- **capturar dados digitados**
    
- **induzir toques/cliques** em botões que a vítima não percebe direito
    

No Android, isso se relaciona com a ideia de **tapjacking**: um app malicioso usa uma sobreposição para fazer a pessoa tocar em controles importantes sem perceber exatamente no que está clicando. A documentação do Android define tapjacking justamente como enganar o usuário por meio de uma interface sobreposta que obscurece a tela ou parte dela.

Então, resumindo de forma bem simples:

**Screen overlay = uma tela falsa sobre a tela verdadeira.**  
O objetivo é **fingir que é o app real** para **roubar dados** ou **manipular a interação da vítima**.

Exemplo fácil:  
você abre o app do banco, mas o malware coloca uma “janela” falsa por cima. Você acha que está digitando no banco, mas está entregando os dados para o invasor.
